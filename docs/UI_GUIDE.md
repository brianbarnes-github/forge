## Field reference (Editor pane)

### Global Settings View вҖ” top-level Config fields

| Field             | Type            | Editable | Maps to Config field   |
|-------------------|-----------------|----------|------------------------|
| Input MIDI        | path label      | no       | `Config::input` (read-only display; set via Open MIDI) |
| Output ABC        | path label      | no       | `Config::output`       |
| Title             | text field      | yes      | `Config::title`        |
| Transcriber       | text field      | yes      | `Config::transcriber`  |
| Tempo (BPM)       | numeric field   | yes      | `Config::tempo`        |
| Global transpose  | numeric field   | yes      | `Config::transpose`    |

### Instruments Table вҖ” list of output ABC parts

| Column   | Source                          |
|----------|---------------------------------|
| X        | `ConfigInstrument::x`           |
| Name     | `ConfigInstrument::name` (LOTRO instrument identifier) |
| Label    | `ConfigInstrument::label` (display name in `T:` header) |
| Sources  | `ConfigInstrument::sources` joined with commas |

### Instrument Detail Form вҖ” the selected row's settings

| Field                    | Control       | Maps to                         |
|--------------------------|---------------|---------------------------------|
| name                     | dropdown      | `ConfigInstrument::name`        |
| label                    | text field    | `ConfigInstrument::label`       |
| transpose semitones      | numeric       | `ConfigInstrument::transposeSemitones` |
| volume %                 | numeric       | `ConfigInstrument::volumePercent` вҖ” adjustment in percent; `0` = no change, `+10` = +10 % louder, `-20` = -20 % quieter |
| drum-map                 | text field    | `ConfigInstrument::drumMap` (only enabled when name is `Drums`) |
| sources                  | checkbox list | `ConfigInstrument::sources` (one checkbox per loaded MIDI track) |

## Field reference (Diagnostics pane)

### Diagnostic List View

| Column   | Source                       |
|----------|------------------------------|
| Severity | `Diagnostic::severity` (Info / Warning / Error) вҖ” coloured dot + label |
| Source   | `Diagnostic::source` (e.g. `RangeConstraint`, `VolumeScale`, `Pipeline`) |
| Tick     | `Diagnostic::tick` (`--` if unset)         |
| Pitch    | `Diagnostic::pitch` (`--` if unset)        |
| Track    | `Diagnostic::trackIndex` (`--` if unset)   |
| Message  | `Diagnostic::message`                       |

### ABC Preview View

- **Editor area** вҖ” read-only monospaced text showing the generated ABC.
- **Status line** вҖ” bytes / bar count / part count.

## Drag-drop targets

The whole Main window is a drag-drop target. Drop:

- `.mid` or `.midi` вҶ’ same as **File вҶ’ Open MIDIвҖҰ**
- `.json`, `.toml`, or `.xml` вҶ’ same as **File вҶ’ Open ConfigвҖҰ**

## Bug-report shorthand

When something doesn't work, please reference the named region:

> "The **Run Converter button** doesn't respond."
> "The **inner splitter** is fixed at 50/50 and won't drag."
> "The **drum-map field** stays disabled even when I select Drums in the **name** dropdown."
> "The **Source checkboxes** aren't appearing for instrument X:2."

That avoids any ambiguity about which of the half-dozen buttons / fields / lists / tables we're talking about.
